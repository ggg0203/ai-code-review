> 2026 年毕业答辩项目 · 全栈开发

---

## 一、项目概述

AI Code Review Platform 是一个基于大语言模型的智能代码审查与项目管理平台。平台以 **FastAPI 后端** 为核心，集成阿里云百炼 DeepSeek-V4-Pro 大模型实现 AI 代码审查和 RAG 知识库问答，前端覆盖 **React Web 管理后台**、**uni-app 移动端** 和 **Three.js 3D 数据大屏** 三种形态。

### 核心价值

- **AI 驱动的代码审查**：SSE 流式实时推送审查结果，覆盖 Bug 风险、性能、安全、规范、最佳实践五个维度，自动评分并持久化
- **RAG 知识库**：将历史审查经验转为 1536 维向量存入 Qdrant，支持语义搜索和 AI 问答
- **多端覆盖**：同一套 API 支撑 Web 后台、移动端 App、3D 数据大屏
- **企业级安全**：JWT 双 Token、RBAC 三级权限、bcrypt 密码哈希

## 三、核心功能详解

### 3.1 AI 代码审查（SSE 流式）

**技术链路**：前端 fetch → FastAPI StreamingResponse → 阿里百炼 deepseek-v4-pro (stream=True) → SSE 逐块推送 → react-markdown 实时渲染

**流程**：
1. 用户粘贴代码片段，选择语言
2. 后端构造结构化 prompt（要求 AI 输出 ## 代码概览 / ## 严重问题 / ## 一般问题 / ## 优化建议 / ## 综合评分表格）
3. 百炼 API 流式返回 token，后端逐块 encode 为 `data: {...}\n\n` 格式的 SSE 事件
4. 前端 ReadableStream 逐块读取，累积文本传给 react-markdown 实时渲染
5. 流结束后，后端自动提取综合评分（正则匹配 综合|**X/10**），写入 Review 记录的 ai_result 和 ai_score 字段

**特色**：
- 首字延迟 < 0.3 秒
- Markdown 富文本渲染（标题/表格/代码块/引用块）
- 评分表格自动高亮（>=80 绿色，>=60 黄色，<60 红色）

### 3.2 RAG 知识库问答

**技术链路**：代码入库 → text-embedding-v2 向量化 (1536 维) → Qdrant 存储 → 语义搜索 → deepseek-v4-pro 生成回答

**两个核心 API**：
- `POST /rag/add`：接收代码/文档文本，向量化后存入 Qdrant（RBAC 限制仅 REVIEWER/ADMIN 可操作）
- `POST /rag/ask`：自然语言提问，Qdrant COSINE 检索 top-k 相似文档片段，拼接为 prompt 上下文，由 LLM 生成回答

### 3.3 3D 数据大屏

- 纯 **Three.js 原生实现**（非 React Three Fiber），TypeScript 编写
- 每 10 秒自动从 `GET /api/stats`（公开接口）拉取平台统计数据
- 球体节点代表项目/审查记录，螺旋分布，连线表示依赖关系
- HTML 覆盖层展示四个统计卡片（活跃项目/审查总数/平均评分/涉及语言）
- OrbitControls 交互支持，星空粒子背景，动态光照 + 节点微动动画

### 3.4 认证与权限

- **JWT 双 Token**：Access Token (30min) + Refresh Token (7d)
- **Refresh 端点**：`POST /auth/refresh` 用 refresh_token 换取新 access_token，带 type 校验防滥用
- **RBAC 三级**：ADMIN (全部权限) / REVIEWER (审查 + 知识库) / DEVELOPER (基本操作)
- **密码安全**：bcrypt 哈希存储，修改密码需验证旧密码

## 四、数据库设计

### 4.1 users 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int (PK) | 自增主键 |
| username | varchar(50) | 用户名，unique |

### 4.2 projects 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int (PK) | 自增主键 |
| name | varchar(100) | 项目名 |
| description | text | 描述 |
| repo_url | varchar(255) | 仓库地址 |
| language | varchar(50) | 主要语言 |
| status | enum(ACTIVE, ARCHIVED, DELETED) | 状态 |
| owner_id | int (FK → users) | 所有者 |

### 4.3 reviews 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int (PK) | 自增主键 |
| pr_title | varchar(200) | PR 标题 |
| code_snippet | text | 待审查代码片段 |
| ai_result | text | AI 审查结果 (Markdown) |
| ai_score | float | AI 评分 (0-100) |
| status | enum(PENDING, REVIEWING, COMPLETED, APPROVED, REJECTED) | 状态 |
| project_id | int (FK → projects) | 关联项目 |
| reviewer_id | int (FK → users) | 审查者 |

## 五、启动方式

### 环境要求
- Docker Desktop（PostgreSQL + Redis + Qdrant）
- Python 3.13 + Node.js 22
- 阿里云百炼 API Key（配置在 server/.env）

### 启动步骤

```bash
# 1. 启动基础设施
cd docker && docker compose up -d

# 2. 启动后端
cd server
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# 3. 启动 Web 前端
cd apps/web
npm install && npm run dev      # http://localhost:5173

# 4. 启动 3D 大屏（可选）
cd apps/screen
npm install && npm run dev      # http://localhost:5174

# 5. 启动移动端（可选）
cd apps/mobile
npm install && npm run dev:mp-weixin
```
